# Codex Working Rules: {{PROJECT_NAME}}

## Default Behavior

- Inspect project docs before coding.
- Keep work scoped to the current task and milestone.
- Prefer existing patterns.
- Verify before reporting done.

## Approval Required

Ask before:

- broad refactors
- dependency additions
- auth, security, billing, data model, migration, deployment, or release changes
- commits, pushes, deploys, publications, destructive commands, or secret rotation

## Evidence Required

Final reports should include:

- changed files
- checks run
- checks not run and why
- security notes
- residual risks
